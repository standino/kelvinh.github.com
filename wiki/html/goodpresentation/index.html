TikZ and PGF

Path

The basic building block of all pictures in Tik Z is the path. A path is a series of straight lines and curves that are connected (that is not the whole picture, but let us ignore the complications for the moment). You start a path by specifying the coordinates of the start position as a point in round brackets, as in (0,0). This is followed by a series of “path extension operations.” The simplest is --, which we used already. It must be followed by another coordinate and it extends the path in a straight line to this new position.

Curved Path

The general syntax for extending a path in a “curved” way is .. controls first control point and second control point .. end point . You can leave out the and second control point , which causes the first one to be used twice.

o Include a line in your document preamble: \usepackage{lgrind} or, if you want the code to include function labels on the right hand side, \usepackage[procnames,noindent]{lgrind}

\begin{lgrind} \input mycode.tex \end{lgrind}
